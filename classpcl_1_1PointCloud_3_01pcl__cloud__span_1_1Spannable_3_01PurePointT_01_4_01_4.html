#include <point_cloud.h>
template<typename PurePointT>
pcl::PointCloud<pcl_cloud_span::Spannable<PurePointT>> class

pcl::PointCloud template specialization that uses span_or_vector for points data instead of std::vector.

Contents

  • Reference
    • Public types
    • Public static functions
    • Constructors, destructors, conversion operators
    • Public functions
    • Public variables

This specialization is a copy of pcl::PointCloud with a couple of changes:

  • extra constructor PointCloud(PointT* data, std::uint32_t width_, std::uint32_t height_ = 1) to initialize point cloud as a span over existing data
  • span_or_vector as container for points instead of std::vector

template<typename PurePointT>
Eigen::Map<Eigen::MatrixXf, Eigen::Aligned, Eigen::OuterStride<>> pcl::PointCloud<pcl_cloud_span::Spannable<PurePointT>><PurePointT>::getMatrixXfMap(int dim, int stride, int offset)

Return an Eigen MatrixXf (assumes float values) mapped to the specified dimensions of the PointCloud.

Parameters
dim in the number of dimensions to consider for each point
stride in the number of values in each point (will be the number of values that separate two of the columns)
offset in the number of dimensions to skip from the beginning of each point (stride = offset + dim + x, where x is the number of dimensions to skip from the end of each point)

Note

This method is for advanced users only! Use with care!

Attention

Compile time flags used for Eigen might affect the dimension of the Eigen::Map returned. If Eigen is using row major storage, the matrix shape would be (number of Points X elements in a Point) else the matrix shape would be (elements in a Point X number of Points). Essentially,

  • Major direction: number of points in cloud
  • Minor direction: number of point dimensions By default, as of Eigen 3.3, Eigen uses Column major storage

Note

for getting only XYZ coordinates out of PointXYZ use dim=3, stride=4 and offset=0 due to the alignment.

Attention

PointT types are most of the time aligned, so the offsets are not continuous!
